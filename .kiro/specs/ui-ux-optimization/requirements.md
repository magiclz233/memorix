# UI/UX 全面优化需求文档

## 1. 项目概述

### 1.1 背景
Lumina 项目当前的前台画廊和后台管理系统在基础功能上已经完善，但在用户体验和交互细节上存在一些可优化的空间。特别是：
- 后台媒体库的筛选功能过于复杂，信息密度高
- 上传中心的设计不适合批量上传场景
- 部分交互细节缺乏视觉反馈和错误处理

### 1.2 目标
通过系统化的 UI/UX 优化，提升整体用户体验，使系统更加易用、高效、美观。

### 1.3 范围
- **后台管理**：媒体库筛选、上传中心、藏品管理、存储管理
- **前台画廊**：照片详情模态框、筛选交互、无限滚动
- **通用优化**：错误处理、加载状态、响应式设计、无障碍性

## 2. 用户故事

### 2.1 后台管理员用户故事

#### US-1: 媒体库高效筛选
**作为** 后台管理员  
**我想要** 快速筛选和定位媒体文件  
**以便** 高效管理大量的图片和视频资源

**验收标准：**
- [ ] 筛选界面简洁清晰，不超过 3 层视觉层级
- [ ] 支持按存储类型、存储实例、发布状态、媒体类型等多维度筛选
- [ ] 筛选条件可以快速重置
- [ ] 筛选结果实时更新，有平滑过渡动画
- [ ] 当前筛选状态清晰可见
- [ ] 支持保存常用筛选组合（可选）

#### US-2: 批量上传体验优化
**作为** 后台管理员  
**我想要** 一次性上传大量图片并清晰了解上传进度  
**以便** 快速完成媒体资源的批量导入

**验收标准：**
- [ ] 支持拖拽多个文件或文件夹上传
- [ ] 上传队列以紧凑的方式展示（如表格或分组卡片）
- [ ] 显示整体上传进度和单个文件进度
- [ ] 支持暂停、恢复、取消单个或全部上传
- [ ] 上传失败的文件可以一键重试
- [ ] 上传完成后有明确的成功提示
- [ ] 大文件分片上传进度平滑，不跳跃

#### US-3: 批量操作简化
**作为** 后台管理员  
**我想要** 简洁高效的批量操作界面  
**以便** 快速处理多个媒体文件

**验收标准：**
- [ ] 批量操作栏信息密度合理，按钮分组清晰
- [ ] 选择模式有明确的开关（如"选择"按钮）
- [ ] 选中的文件数量清晰显示
- [ ] 批量删除、下载、发布等操作有进度反馈
- [ ] 支持全选当前页和清除选择
- [ ] 危险操作（如删除）有二次确认

#### US-4: 藏品管理优化
**作为** 后台管理员  
**我想要** 直观的拖拽排序和批量管理功能  
**以便** 高效组织藏品中的媒体内容

**验收标准：**
- [ ] 拖拽排序时有清晰的视觉反馈（拖拽预览、占位符）
- [ ] 支持批量选择和移除
- [ ] 添加媒体的 Picker 对话框响应式适配
- [ ] 空状态有引导性的设计和动画
- [ ] 排序变更自动保存，有保存成功提示

#### US-5: 存储管理改进
**作为** 后台管理员  
**我想要** 清晰的存储扫描进度和状态反馈  
**以便** 了解存储同步的实时情况

**验收标准：**
- [ ] 扫描操作有实时进度条和日志查看
- [ ] 卡片视图和列表视图信息展示一致
- [ ] 依赖警告对话框文案友好，说明影响范围
- [ ] 缓存清理流程简化，显示预估清理空间
- [ ] 存储状态指示器清晰（在线、扫描中、禁用）

### 2.2 前台访客用户故事

## 3. 功能需求

### 3.1 媒体库筛选重设计

#### FR-1.1 筛选界面结构
**优先级：** 高

**需求描述：**
重新设计媒体库筛选界面，采用渐进式展开的方式，减少初始视觉复杂度。

**详细要求：**
1. **一级筛选（始终可见）**
   - 存储类型快速切换（All / Local / NAS / S3）
   - 搜索框（按文件名搜索）
   - 视图切换（网格/列表）
   - 高级筛选按钮

2. **二级筛选（点击展开）**
   - 存储实例选择（当选择具体存储类型时显示）
   - 发布状态（已发布/未发布/全部）
   - 媒体类型（图片/视频/动图/全部）
   - 日期范围选择器
   - Hero 标记筛选

3. **筛选状态指示**
   - 显示当前激活的筛选条件数量
   - 支持快速清除所有筛选
   - 筛选条件以 Chip 形式展示，可单独移除

#### FR-1.2 筛选交互优化
**优先级：** 高

**需求描述：**
优化筛选交互，提供即时反馈和平滑过渡。

**详细要求：**
1. 筛选变化时使用骨架屏而非完整 Skeleton
2. 筛选结果数量实时显示
3. 筛选历史记录（最近 5 次）
4. URL 参数同步，支持分享筛选结果
5. 筛选预设保存（可选功能）

### 3.2 上传中心重构

#### FR-2.1 批量上传界面设计
**优先级：** 高

**需求描述：**
重新设计上传中心，采用紧凑的表格或分组卡片布局，适配批量上传场景。

**详细要求：**
1. **整体布局**
   - 左侧：上传配置区（存储选择、上传选项）
   - 右侧：上传队列区（表格或紧凑卡片）
   - 顶部：整体进度条和统计信息

2. **上传队列展示**
   - 表格模式：文件名、大小、进度、状态、操作
   - 分组模式：按状态分组（上传中、等待中、已完成、失败）
   - 支持折叠/展开分组
   - 每组显示汇总信息（文件数、总大小、平均速度）

3. **进度反馈**
   - 整体进度条（顶部）
   - 单个文件进度条（行内或卡片内）
   - 上传速度和剩余时间估算
   - 成功/失败数量统计

#### FR-2.2 上传控制功能
**优先级：** 高

**需求描述：**
提供完整的上传控制功能，包括暂停、恢复、取消、重试。

**详细要求：**
1. **全局控制**
   - 暂停全部上传
   - 恢复全部上传
   - 取消全部上传
   - 清除已完成项
   - 重试全部失败项

2. **单文件控制**
   - 暂停/恢复单个文件
   - 取消单个文件
   - 重试单个文件
   - 从队列中移除

3. **智能上传**
   - 并发上传数量控制（默认 3 个）
   - 失败自动重试（最多 3 次）
   - 网络断开时自动暂停
   - 网络恢复时提示恢复上传

#### FR-2.3 分片上传优化
**优先级：** 中

**需求描述：**
优化大文件分片上传的进度计算，使进度条更平滑。

**详细要求：**
1. 进度计算包含哈希计算阶段（0-10%）
2. 分片上传阶段平滑递增（10-95%）
3. 完成阶段（95-100%）
4. 显示当前阶段文本（"计算哈希中" / "上传中" / "完成中"）

### 3.3 照片详情模态框优化

#### FR-3.1 编辑模式改进
**优先级：** 中

**需求描述：**
优化 Frame 模式下的元数据编辑体验，使编辑功能更明确。

**详细要求：**
1. 添加"编辑"按钮切换编辑模式
2. 编辑模式下，输入框有明显的边框和背景
3. 保存按钮在有修改时始终可见
4. 支持 Ctrl+S / Cmd+S 快捷键保存
5. 未保存时切换照片有提示确认

#### FR-3.2 导航优化
**优先级：** 中

**需求描述：**
优化照片切换动画和缩略图滚动体验。

**详细要求：**
1. 简化切换动画逻辑，使用统一的滑动效果
2. 缩略图滚动条优化，添加拖拽指示
3. 当前照片在缩略图中自动居中
4. 支持键盘快捷键（左右箭头、Home、End）
5. 移动端支持滑动手势

### 3.4 错误处理统一化

#### FR-4.1 Toast 通知系统
**优先级：** 高

**需求描述：**
实现统一的 Toast 通知系统，替代所有 `alert()` 和 `confirm()` 调用。

**详细要求：**
1. 支持 4 种类型：success、error、warning、info
2. 支持自定义持续时间
3. 支持操作按钮（如"重试"、"查看详情"）
4. 支持堆叠显示多个通知
5. 移动端适配，位置和尺寸优化

#### FR-4.2 错误边界
**优先级：** 中

**需求描述：**
实现全局错误边界，捕获组件错误并友好展示。

**详细要求：**
1. 捕获组件渲染错误
2. 显示友好的错误页面
3. 提供"重新加载"按钮
4. 错误信息上报（可选）
5. 开发环境显示详细错误堆栈

#### FR-4.3 网络错误处理
**优先级：** 中

**需求描述：**
统一处理网络请求错误，提供重试机制。

**详细要求：**
1. 网络超时自动重试（最多 3 次）
2. 401 错误自动跳转登录
3. 403 错误显示权限不足提示
4. 404 错误显示资源不存在
5. 500 错误显示服务器错误，提供重试按钮

### 3.5 加载状态优化

#### FR-5.1 统一加载指示器
**优先级：** 中

**需求描述：**
统一所有加载状态的视觉语言，提供一致的用户体验。

**详细要求：**
1. 全局加载进度条（顶部）
2. 局部加载使用 Spinner 或 Skeleton
3. 按钮加载状态（禁用 + Spinner）
4. 长时间加载显示进度百分比
5. 加载超时提示

#### FR-5.2 骨架屏优化
**优先级：** 低

**需求描述：**
优化骨架屏设计，使其更接近实际内容布局。

**详细要求：**
1. 画廊骨架屏使用 Masonry 布局
2. 媒体库骨架屏匹配网格列数
3. 骨架屏动画流畅（脉冲或渐变）
4. 支持 Dark Mode

### 3.6 响应式设计改进

#### FR-6.1 移动端优化
**优先级：** 中

**需求描述：**
优化移动端体验，确保所有功能在小屏幕上可用。

**详细要求：**
1. 触摸目标至少 44x44px
2. 筛选使用抽屉式面板
3. 批量操作使用底部操作栏
4. 表格在移动端转为卡片列表
5. 导航使用汉堡菜单

#### FR-6.2 横屏模式
**优先级：** 低

**需求描述：**
优化横屏模式下的布局，充分利用屏幕空间。

**详细要求：**
1. 画廊使用更多列数
2. 照片详情模态框优化布局
3. 后台管理侧边栏自动收起

### 3.7 无障碍性改进

#### FR-7.1 键盘导航
**优先级：** 中

**需求描述：**
实现完整的键盘导航支持，确保所有功能可通过键盘访问。

**详细要求：**
1. Tab 键顺序合理
2. 焦点样式清晰可见
3. 支持常用快捷键（Ctrl+S、ESC、Enter）
4. 模态框打开时焦点管理
5. 下拉菜单支持方向键导航

#### FR-7.2 ARIA 属性
**优先级：** 低

**需求描述：**
为所有交互元素添加适当的 ARIA 属性，提升屏幕阅读器体验。

**详细要求：**
1. 按钮添加 `aria-label`
2. 加载状态添加 `aria-busy`
3. 错误提示添加 `aria-live`
4. 模态框添加 `role="dialog"`
5. 表单字段添加 `aria-describedby`

## 4. 非功能需求

### 4.1 性能要求
- 筛选响应时间 < 300ms
- 上传队列渲染 1000 个文件无卡顿
- 照片切换动画 60fps
- 首屏加载时间 < 2s

### 4.2 兼容性要求
- 支持 Chrome、Firefox、Safari、Edge 最新两个版本
- 支持 iOS 14+ 和 Android 10+
- 支持触摸和鼠标交互
- 支持键盘导航

### 4.3 可访问性要求
- 符合 WCAG 2.1 AA 级别
- 颜色对比度 ≥ 4.5:1
- 所有功能可通过键盘访问
- 支持屏幕阅读器

### 4.4 可维护性要求
- 组件化设计，复用性高
- 代码注释完整
- 遵循项目现有的设计规范
- 国际化支持（中英文）

---

## 5. 约束条件

### 5.1 技术约束
- 必须使用 Next.js 15+ 和 React 19
- 必须使用 Shadcn UI 组件库
- 必须保持 Light/Dark 双模态支持
- 必须使用 TypeScript

### 5.2 设计约束
- 必须遵循 Lumina Pro 设计风格
- 必须使用 zinc 色系和 indigo 主色
- 必须保持极简杂志风（Light）和流光黑洞风（Dark）
- 禁止使用纯黑背景

### 5.3 业务约束
- 不能破坏现有功能
- 必须向后兼容现有数据
- 必须保持 API 接口不变
- 优化必须渐进式实施

## 8. 风险与依赖

### 8.1 风险
- **高风险**：上传中心重构可能影响现有上传逻辑
- **中风险**：筛选重设计可能需要调整后端 API
- **低风险**：动画优化可能影响性能

### 8.2 依赖
- 依赖 Shadcn UI 组件库的更新
- 依赖 Framer Motion 动画库
- 依赖现有的 Server Actions 和 API

### 8.3 缓解措施
- 充分的单元测试和集成测试
- 渐进式发布，先在测试环境验证
- 保留旧版本的回滚能力
- 详细的变更文档和迁移指南
